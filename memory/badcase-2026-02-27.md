## 系统性问题分析

### 根本原因
当前工作流存在**断点**：
```
日报生成 → [断点] → 汇总页更新 → [断点] → 部署
```

日报生成后，汇总页不会自动感知新文件，需要：
1. 手动运行更新脚本，或
2. 等待定时任务（可能已过期）

### 建议改进方案

**方案A：合并任务（推荐）**
在日报生成任务的最后一步，自动调用汇总页更新逻辑：
```bash
# 在日报生成脚本末尾添加
python update_index.py  # 自动扫描文件并更新index.html
```

**方案B：文件监听**
使用文件系统监听，当新HTML文件生成时自动触发更新：
```bash
# 使用 inotifywait 或类似工具
inotifywait -m -e create *.html --format '%w%f' | \
  while read file; do update_index.py; done
```

**方案C：动态生成**
将 index.html 改为服务端渲染或客户端JS动态加载，无需手动维护：
```javascript
// 前端自动扫描目录
fetch('/api/list-files').then(files => renderCards(files));
```
